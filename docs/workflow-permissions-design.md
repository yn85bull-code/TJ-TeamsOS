# TeamOS 課題・タスク・承認フロー設計

## 基本方針

TeamOSでは、課題・タスク・承認フローを以下の流れで管理する。

```text
登録
↓
タスク化
↓
進捗報告
↓
承認申請
↓
Manager確認
↓
Owner / Admin承認
↓
完了
```

目的は、現場の実行状況を可視化しつつ、最終判断・承認は経営側で統制すること。

## 権限定義

### Owner

対象者は代表のみ。全データの閲覧・作成・編集・削除、すべての承認、ユーザー権限変更、TeamOS上のすべての機能操作が可能。

### Admin

対象者は経営陣のみ。全データの閲覧・作成・編集、すべての承認、Ownerと同等の全体管理が可能。

制限として、ユーザー権限の変更とOwner専用操作は不可。

### Manager

対象者は拠点責任者・部門責任者。

権限:

- 自部門・自拠点の課題・タスクを閲覧可能
- 自分が登録者または担当者になっているタスクのみ編集可能
- 自分が登録者または担当者になっているタスクのみ進捗更新可能
- 自分が登録者または担当者になっているタスクのみ承認申請可能
- 自部門・自拠点に関するタスクの確認が可能

制限:

- 最終承認は不可
- 他部門・他拠点の課題・タスクは閲覧不可
- 自分が関係していないタスクの進捗更新・承認申請は不可
- 権限変更は不可

Managerは現場責任者として確認までは可能だが、最終承認はできない。

### Member

対象者は営業・事務などの一般社員。

権限:

- 自分が登録者になっているタスクのみ閲覧・編集可能
- 自分が担当者になっているタスクのみ閲覧・編集可能
- 自分が担当者または登録者になっているタスクのみ進捗更新可能
- 自分が担当者または登録者になっているタスクのみ承認申請可能
- TaurosAIのナレッジ機能を利用可能

制限:

- 他人のタスクは閲覧不可
- 他部門・他拠点の課題は閲覧不可
- 承認一覧は閲覧不可
- 最終承認は不可
- 確認は不可
- レポート、設定、ユーザー管理、権限変更は不可

## 共通ルール

Owner / Admin以外は、自分が担当者または登録者ではないタスクに対して、以下の操作はできない。

- 進捗更新
- 編集
- 承認申請
- 完了報告

権限外の情報は画面上で非表示にし、API側でも取得・更新できないように制御する。

## フロー詳細

### 登録

課題、依頼、やることを登録する段階。タイトル、内容、カテゴリ、優先度、部門、拠点、登録者、担当責任者、担当者、期限、添付ファイル、関連メモを設定する。

### タスク化

登録された課題や依頼を、実行可能なタスクに分解する段階。誰が・何を・いつまでに行うかを明確にする。

### 進捗報告

担当者が作業状況を報告する段階。Owner / Admin以外は、自分が担当者または登録者になっているタスクのみ進捗報告できる。

### 承認申請

担当者または登録者が、対応完了後に確認・承認を依頼する段階。Owner / Admin以外は、自分が担当者または登録者になっているタスクのみ承認申請できる。

### Manager確認

Manager確認は原則必須。Managerは自部門・自拠点に関するタスクのみ確認可能。確認は最終承認ではなく、確認後にOwner / Adminの承認待ちへ進む。

Owner / AdminはManager確認なしで直接承認可能。

### Owner / Admin承認

最終承認はOwner / Adminのみ可能。ManagerとMemberは最終承認不可。

## 差し戻しルール

差し戻しが発生した場合は、担当責任者に戻す。差し戻し理由の入力を必須にする。

記録項目:

- 差し戻し理由
- 差し戻し者
- 差し戻し日時
- 再対応期限
- コメント

## 承認済み後のルール

承認済みデータは履歴として残す。原則として承認済み後の編集は不可。

承認済み後に変更が必要な場合は、再申請または新規作成で対応する。承認済みデータの削除も原則不可。削除が必要な場合はOwnerのみ可能。

## 推奨ステータス

```text
登録済み
タスク化済み
未着手
進行中
進捗報告済み
承認申請中
確認待ち
確認済み
差し戻し
承認済み
完了
```

## 実装時の注意点

- 権限ごとにサイドバー表示を制御する
- 権限ごとにAPI取得範囲を制御する
- Memberは自分関連タスクのみ表示する
- Managerは自部門・自拠点の閲覧は可能だが、更新は自分関連のみとする
- Owner / Admin以外は権限変更できない
- Adminは全閲覧・全編集可能だが、権限変更はできない
- Manager確認は原則必須
- Owner / Adminは確認なしで直接承認可能
- 差し戻し時は理由入力を必須にする
- 承認済み後は編集不可
- 承認済みデータは削除不可
- 削除が必要な場合はOwnerのみ可能
- 承認済み後の変更は再申請または新規作成で対応する
